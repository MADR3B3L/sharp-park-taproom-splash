# GitHub Pages Deploy Notes

## Fast path

1. Create a GitHub repository for this site.
2. Push this folder to that repository.
3. Turn on GitHub Pages from the `main` branch root.
4. Confirm the site works on the default `github.io` URL first.
5. After that, connect the custom domain from GoDaddy.

## Why this path

- free static hosting
- easy to keep updating
- no forced rebuild inside GoDaddy Websites + Marketing
- easy to reuse for future client sites

## Before custom domain

Make sure these work:

- homepage
- menu page
- assets load correctly
- tap data renders
- mobile layout is acceptable

## Recommended post-launch follow-up

- add a mobile polish pass
- connect real calendar embed
- review image weights and compress large branding assets if needed
- decide whether `site-options.html` stays public or becomes an owner-only tool
